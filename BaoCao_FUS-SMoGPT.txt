4. THIẾT KẾ LỚP (CLASS DESIGN) - KỸ THUẬT OOP
================================================================================

4.1. Interface - IUser (Models/IUser.cs)
-----
Interface định nghĩa hợp đồng cho tất cả người dùng:

    public interface IUser
    {
        string Username { get; set; }
        string Password { get; set; }
        UserRole Role { get; }
        bool HasPermission(string action);
        string GetDisplayInfo();
    }

Mục đích: Đảm bảo mọi người dùng (Admin, Student) đều phải implement
các method này → loose coupling, dễ mở rộng.

4.2. Abstract Class - Person (Models/Person.cs)
-----
Lớp trừu tượng - không thể tạo instance trực tiếp:

    public abstract class Person : IUser
    {
        // Fields (private) - đóng gói dữ liệu
        private string _name;
        private string _id;
        
        // Properties - truy cập có kiểm soát
        public string Name { get; set; }  // Có validation
        public string Id { get; set; }    // Có validation
        
        // Abstract method - BẮT BUỘC lớp con override
        public abstract string GetDisplayInfo();
        
        // Virtual method - lớp con CÓ THỂ override
        public virtual bool HasPermission(string action) => false;
    }

Kỹ thuật sử dụng:
- Abstract class: Person không thể new Person()
- Fields (private): _name, _id - đóng gói
- Properties: Name, Id - truy cập có validation
- Abstract method: GetDisplayInfo() - đa hình bắt buộc
- Virtual method: HasPermission() - đa hình tùy chọn
- Interface implementation: IUser

4.3. Kế thừa - Student : Person (Models/Student.cs)
-----
    public class Student : Person
    {
        // Thêm properties riêng
        public string Mssv { get; set; }
        public string NganhHoc { get; set; }
        public string KhoaHoc { get; set; }
        public double Gpa { get; set; }       // Validation: 0-4
        public HocLuc XepLoaiHocLuc { get; }  // Computed property
        
        // ĐA HÌNH: Override abstract method
        public override string GetDisplayInfo() 
            => $"Sinh viên: {Name}\nMSSV: {Mssv}...";
        
        // ĐA HÌNH: Override virtual method
        public override bool HasPermission(string action)
            => action == "viewSelf";  // Chỉ có quyền xem bản thân
    }

4.4. Kế thừa - Admin : Person (Models/Admin.cs)
-----
    public class Admin : Person
    {
        // ĐA HÌNH: Override khác Student
        public override string GetDisplayInfo()
            => "Quản trị viên...";
        
        // ĐA HÌNH: Admin có TẤT CẢ quyền
        public override bool HasPermission(string action)
            => true;  // Toàn quyền
    }

So sánh đa hình (Polymorphism):
- Student.HasPermission("delete") → false (không có quyền)
- Admin.HasPermission("delete") → true (có quyền)
→ Cùng một method, hai lớp con cho kết quả khác nhau = ĐA HÌNH

4.5. Sơ đồ kế thừa:
-----
    IUser (Interface)
      ↑ implements
    Person (Abstract class)
      ↑ kế thừa          ↑ kế thừa
    Student              Admin

    IDataService (Interface)
      ↑ implements
    JsonDataService

    IAuthService (Interface)
      ↑ implements
    AuthService

5.1. Thuật toán xếp loại học lực (GPA Classification)
-----
Input: Điểm GPA (double, 0.0 - 4.0)
Output: Xếp loại học lực (HocLuc enum)

Lưu đồ:
    START
      ↓
    Nhận giá trị GPA
      ↓
    GPA == 4.0? → CÓ → Xuất Sắc → END
      ↓ KHÔNG
    GPA >= 3.5? → CÓ → Giỏi → END
      ↓ KHÔNG
    GPA >= 2.5? → CÓ → Khá → END
      ↓ KHÔNG
    GPA >= 2.0? → CÓ → Trung Bình → END
      ↓ KHÔNG
    GPA >= 1.0? → CÓ → Yếu → END
      ↓ KHÔNG
    Kém → END

Code (C# switch expression):
    return _gpa switch
    {
        4.0 => HocLuc.XuatSac,
        >= 3.5 => HocLuc.Gioi,
        >= 2.5 => HocLuc.Kha,
        >= 2.0 => HocLuc.TrungBinh,
        >= 1.0 => HocLuc.Yeu,
        _ => HocLuc.Kem
    };

Độ phức tạp: O(1) - constant time vì sử dụng switch expression

5.2. Thuật toán xác thực đăng nhập (Authentication)
-----
Input: username (string), password (string)
Output: IUser object hoặc null

Lưu đồ:
    START
      ↓
    Nhận username, password
      ↓
    Username rỗng? → CÓ → return null → END
      ↓ KHÔNG
    username == "admin" VÀ password == "123"?
      ↓ CÓ → return Admin object → END
      ↓ KHÔNG
    Duyệt danh sách sinh viên:
      Tìm sinh viên có MSSV == username VÀ password khớp?
      ↓ CÓ → return Student object → END
      ↓ KHÔNG TÌM THẤY
    return null (đăng nhập thất bại) → END

Code:
    public IUser? Authenticate(string username, string password)
    {
        // Bước 1: Kiểm tra Admin
        if (username == "admin" && password == "123")
            return new Admin("Quản trị viên", "admin", "123");
        
        // Bước 2: Tìm trong danh sách sinh viên
        foreach (var student in students)
        {
            if (student.Mssv == username && student.Password == password)
                return student;
        }
        
        return null; // Thất bại
    }

Độ phức tạp: O(n) - linear search qua n sinh viên

5.3. Thuật toán tìm kiếm sinh viên (Fuzzy Search)
-----
Input: keyword (string)
Output: List<Student> phù hợp

Thuật toán: Tìm kiếm mờ trên nhiều trường
- So khớp không phân biệt hoa/thường (case-insensitive)
- Tìm trên: Tên, MSSV, Ngành học, Khóa học, Học lực

Code (sử dụng LINQ):
    return students.Where(s =>
        s.Name.ToLower().Contains(searchTerm) ||
        s.Mssv.ToLower().Contains(searchTerm) ||
        s.NganhHoc.ToLower().Contains(searchTerm) ||
        s.KhoaHoc.ToLower().Contains(searchTerm) ||
        s.HocLucDisplay.ToLower().Contains(searchTerm)
    ).ToList();

Độ phức tạp: O(n * m) với n = số sinh viên, m = độ dài chuỗi tìm kiếm

5.4. Thuật toán CRUD (Create, Read, Update, Delete)
-----
a) Create (Thêm sinh viên):
   - Validate dữ liệu đầu vào
   - Kiểm tra trùng MSSV → DuplicateStudentException
   - Thêm vào List và lưu file JSON

b) Read (Đọc):
   - Đọc file JSON → Deserialize thành List<Student>
   - Linear search theo MSSV → O(n)

c) Update (Cập nhật):
   - Tìm index của sinh viên cần sửa → O(n)
   - Thay thế object tại index đó
   - Lưu lại file JSON

d) Delete (Xóa):
   - Tìm sinh viên theo MSSV → O(n)
   - Remove khỏi List
   - Lưu lại file JSON

7. GIẢI THÍCH CODE CHI TIẾT
================================================================================

7.1. BaseViewModel - Nền tảng MVVM
-----
File: ViewModels/BaseViewModel.cs

Implements INotifyPropertyChanged - giao diện WPF cần interface này
để tự động cập nhật khi dữ liệu thay đổi.

    protected bool SetProperty<T>(ref T field, T value, 
        [CallerMemberName] string? propertyName = null)
    {
        if (Equals(field, value)) return false;  // Không thay đổi → bỏ qua
        field = value;                            // Gán giá trị mới
        OnPropertyChanged(propertyName);          // Thông báo UI cập nhật
        return true;
    }

Giải thích:
- Generic method <T>: hoạt động với mọi kiểu dữ liệu
- ref T field: truyền tham chiếu đến backing field
- [CallerMemberName]: tự động lấy tên property gọi method
- Equals check: tránh cập nhật UI không cần thiết

7.2. RelayCommand - Command Pattern
-----
File: ViewModels/RelayCommand.cs

Cho phép bind button click từ XAML vào method trong ViewModel:

    XAML: <Button Command="{Binding LoginCommand}"/>
    ViewModel: LoginCommand = new RelayCommand(ExecuteLogin);

Gồm 2 phần:
- Execute: action được thực thi khi click
- CanExecute: điều kiện để button enabled/disabled

7.3. JsonDataService - Đọc/Ghi JSON
-----
File: Services/JsonDataService.cs

Sử dụng System.Text.Json để serialize/deserialize:
- LoadData(): Đọc students.json → List<Student>
- SaveChanges(): List<Student> → ghi ra students.json

Exception handling:
- JsonException: khi file JSON bị hỏng format
- IOException: khi không thể đọc/ghi file
- FileNotFoundException: khi file chưa tồn tại → tạo data mẫu

7.4. Data Binding trong XAML
-----
WPF Data Binding kết nối ViewModel ↔ View tự động:

    <!-- TextBox bind đến property Username trong ViewModel -->
    <TextBox Text="{Binding Username, UpdateSourceTrigger=PropertyChanged}"/>
    
    <!-- Button bind đến LoginCommand -->
    <Button Command="{Binding LoginCommand}"/>
    
    <!-- DataGrid bind đến danh sách Students -->
    <DataGrid ItemsSource="{Binding Students}"/>

UpdateSourceTrigger=PropertyChanged: cập nhật ngay khi gõ (không đợi focus out)

7.5. GPA Computed Property
-----
File: Models/Student.cs

Property XepLoaiHocLuc là computed (chỉ có getter, không có setter).
Giá trị được tính tự động từ GPA mỗi khi truy cập:

    [JsonIgnore]  // Không lưu vào JSON (tính lại mỗi lần)
    public HocLuc XepLoaiHocLuc
    {
        get => _gpa switch
        {
            4.0 => HocLuc.XuatSac,
            >= 3.5 => HocLuc.Gioi,
            // ...
        };
    }

[JsonIgnore]: Thuộc tính này không serialize vào JSON vì được tính từ GPA.

8. XỬ LÝ NGOẠI LỆ (EXCEPTION HANDLING)
================================================================================

8.1. Custom Exceptions (3 lớp):

a) StudentNotFoundException:
   - Khi: Tìm sinh viên bằng MSSV nhưng không có
   - Nơi dùng: GetStudentByMssv(), DeleteStudent(), UpdateStudent()
   - Xử lý: Hiển thị MessageBox thông báo

b) DuplicateStudentException:
   - Khi: Thêm sinh viên với MSSV đã tồn tại
   - Nơi dùng: AddStudent()
   - Xử lý: Hiển thị warning trong form dialog

c) InvalidGpaException:
   - Khi: GPA nhập vào ngoài khoảng 0-4
   - Nơi dùng: Student.Gpa setter, StudentFormViewModel
   - Xử lý: Hiển thị lỗi trong form

8.2. Exception Handling Patterns:

a) Try-Catch trong ViewModel:
    private void ExecuteLogin(object? parameter)
    {
        try
        {
            IUser? user = _authService.Authenticate(Username, Password);
            if (user != null) LoginSuccessful?.Invoke(user);
            else ErrorMessage = "Sai tên đăng nhập...";
        }
        catch (Exception ex)
        {
            ErrorMessage = $"Đã xảy ra lỗi: {ex.Message}";
        }
    }

b) Try-Catch trong Service:
    private void LoadData()
    {
        try { /* đọc file */ }
        catch (JsonException ex) { /* file bị hỏng → tạo data mẫu */ }
        catch (IOException ex) { /* không đọc được → list rỗng */ }
    }

c) Re-throw Pattern:
    catch (DuplicateStudentException)
    {
        throw;  // Ném lại cho caller xử lý
    }

9. LUỒNG HOẠT ĐỘNG (APPLICATION FLOW)
================================================================================

9.1. Luồng khởi động:
    App.xaml.cs → Startup
      ↓
    Tạo JsonDataService (đọc students.json)
      ↓
    Tạo AuthService (inject DataService)
      ↓
    Tạo LoginViewModel (inject AuthService)
      ↓
    Hiển thị LoginWindow

9.2. Luồng đăng nhập Admin:
    Nhập "admin" / "123"
      ↓
    LoginCommand.Execute()
      ↓
    AuthService.Authenticate("admin", "123")
      ↓
    Trả về Admin object
      ↓
    LoginSuccessful event → MainWindow mở
      ↓
    MainViewModel.CurrentUser = Admin
      ↓
    LoadStudents() → hiển thị TẤT CẢ sinh viên
      ↓
    IsAdmin = true → hiện nút CRUD

9.3. Luồng đăng nhập Sinh viên:
    Nhập "SE180001" / "1"
      ↓
    AuthService.Authenticate("SE180001", "1")
      ↓
    Tìm thấy Student SE180001
      ↓
    MainWindow mở
      ↓
    LoadStudents() → chỉ hiển thị BẢN THÂN
      ↓
    IsAdmin = false → ẨN nút CRUD

9.4. Luồng thêm sinh viên:
    Admin click "➕ Thêm"
      ↓
    RequestStudentForm event → mở StudentFormDialog
      ↓
    Nhập thông tin: Tên, MSSV, Ngành, Khóa, GPA
      ↓
    GPA thay đổi → Preview học lực realtime
      ↓
    Click "💾 Lưu"
      ↓
    Validate → Tạo Student object
      ↓
    SaveCompleted event → MainViewModel.AddNewStudent()
      ↓
    JsonDataService.AddStudent() → kiểm tra trùng MSSV
      ↓
    Lưu vào students.json → LoadStudents() → cập nhật DataGrid

9.5. Luồng xóa sinh viên:
    Admin chọn sinh viên trong bảng → click "🗑️ Xóa"
      ↓
    MessageBox xác nhận
      ↓
    "Yes" → JsonDataService.DeleteStudent()
      ↓
    Xóa khỏi list → SaveChanges() → LoadStudents()

9.6. Luồng đăng xuất:
    Click "🚪 Đăng xuất"
      ↓
    RequestLogout event
      ↓
    Tạo LoginWindow mới → đóng MainWindow

   - Sửa: Chọn sinh viên → Click "✏️ Sửa" → sửa form → Lưu
   - Xóa: Chọn sinh viên → Click "🗑️ Xóa" → Xác nhận
   - Chi tiết: Click vào dòng sinh viên → panel bên phải hiển thị
